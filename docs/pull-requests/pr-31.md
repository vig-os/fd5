# [PR 31](https://github.com/vig-os/fd5/pull/31) feat(h5io): implement dict_to_h5 and h5_to_dict metadata helpers

## Description

Implement the `fd5.h5io` module with `dict_to_h5` and `h5_to_dict` for lossless round-trip conversion between Python dicts and HDF5 groups/attrs. This is the foundation of all metadata I/O in fd5.

## Changes Made

**`src/fd5/h5io.py`** â€” New module (105 lines) with two public functions:
- `dict_to_h5(group, d)` â€” writes nested dicts as HDF5 groups with attrs
- `h5_to_dict(group)` â€” reads groups/attrs back to dicts

Type mapping follows [white-paper.md Â§ Implementation Notes](white-paper.md#h5_to_dict--dict_to_h5-type-mapping):
- `str` â†’ UTF-8 attr, `int` â†’ int64 attr, `float` â†’ float64 attr, `bool` â†’ numpy.bool_ attr
- `list[int|float]` â†’ numpy array attr, `list[str]` â†’ vlen string array attr, `list[bool]` â†’ numpy bool array attr
- `dict` â†’ sub-group (recursive), `None` â†’ skipped (absent attr)
- Keys written in sorted order for deterministic layout (critical for hashing)
- `h5_to_dict` reads only attrs, never datasets
- Unsupported types raise `TypeError`

**`tests/test_h5io.py`** â€” 38 tests covering:
- Scalar types (str, int, float, bool)

- Error handling (TypeError on unsupported types)

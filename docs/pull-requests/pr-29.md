# [PR 29](https://github.com/vig-os/fd5/pull/29) spike: validate h5py streaming chunk write + inline hashing (#24)

## Summary

- Add proof-of-concept script (`scripts/spike_chunk_hash.py`) that tests two h5py approaches for inline SHA-256 hashing during chunked file creation: `write_direct_chunk()` and standard chunked writes with pre-hash.
- Measures SHA-256 overhead (~31% on 1 MiB chunks, throughput >260 MiB/s) and verifies data integrity via read-back hash comparison.
- Findings documented as a [comment on #24](https://github.com/vig-os/fd5/issues/24#issuecomment-3956173124): recommends `write_direct_chunk()` for the `ChunkHasher` in #14.

Closes #24
